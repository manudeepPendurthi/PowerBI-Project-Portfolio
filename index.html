IoT Temporal Data

  • The IoT devices installed on the on-field devices are programmed to transmit the timely data of various on-field parameters to the AWS IoT Core. The data aggregated at the AWS IoT core is stored in a PostgreSQL table using AWS Lambda. A connection between the PostgreSQL database and Power BI is established to develop the reports.
  • The first page contains the summary of the temporal data visualising the Geo-Location of the asset using a map visual and maximum values of the on-field parameters using gauge visuals. The asset model card will be displaying the manufacturer name and the model of the asset. The asset ID slicer will allow the user to select the desired asset to visualise the data.
  • Upon clicking the temporal data button the user will be navigated to the second page which shows the temporal graph of the on-field parameters. When hovered on any of the data points the user will be presented with the tooltips providing the extra information about the assets.


      • Job Observation Report

      • I have developed an algorithm using machine vision techniques to read through the handwritten job observation forms produced by the on-site employees to report the incidents at the jobsite. The algorithm converts the handwritten text into digital format which will be stored in a PostgreSQL database. The aggregated data from the PostgreSQL database is then connected to Power BI to develop the reports.
      • The first page contains the summary of the reported incidents and observations. The top of the page has the necessary slicers to filter the incidents using various attributes. The right of the report page has multiple cards showing the sum of the reported incidents based on different filters. The % view button will switch the visuals on the graphs from count view to % view and vice versa.
      • Upon clicking the detailed report button the used will be navigated to a tabular representation of the data with which the user can read the actual observations written by the employees.



          • Project Work Package Report

          • The information about various work packages in a project is stored in a MySQL database using a proprietary data capturing tool. The aggregated data from the database is connected to Power BI to develop reports.
          • The first page visualises the summary of the work packages that are filtered by the activity type.
          • The second and third pages visualise the timeline trends and the detailed summary of the work packages respectively.
